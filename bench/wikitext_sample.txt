The history of computing is a story of abstraction. Each generation of engineers built a layer atop the last, hiding the complexity below so that the next generation could reason about larger problems. The transistor gave way to the logic gate, the gate to the arithmetic unit, the unit to the processor, and the processor to the operating system. By the time a programmer writes a single line of code in a high level language, dozens of layers of abstraction stand between that line and the electrons moving through silicon.

Machine learning extends this tradition. A neural network is, at its core, a sequence of matrix multiplications interleaved with simple nonlinear functions. Yet from this modest machinery emerge systems that translate languages, recognize faces, and hold conversations. The transformer architecture, introduced in 2017, replaced the recurrent connections of earlier models with a mechanism called attention. Attention allows every position in a sequence to consider every other position directly, without passing information step by step through a chain of hidden states.

The cost of this flexibility is computation. A naive attention layer compares each token to every other token, so its work grows with the square of the sequence length. For short prompts this is negligible, but for long documents it becomes the dominant expense. Engineers have responded with a variety of techniques: caching the keys and values of past tokens so they need not be recomputed, grouping query heads so that several share a single set of keys, and writing specialized kernels that exploit the memory hierarchy of the graphics processor.

Quantization is one of the most effective of these techniques. A model trained in sixteen bit floating point can often be served in eight bits, or even four, with little loss in quality. The idea is simple: instead of storing each weight as a wide floating point number, store a small integer together with a scaling factor that maps it back to the original range. The arithmetic is then carried out in low precision, saving both memory and bandwidth. The challenge lies in choosing the scales so that the most important weights are represented faithfully while the rest are compressed aggressively.

Inference serving brings its own concerns. A production system must handle many requests at once, each at a different stage of generation. A scheduler admits new requests, advances each by one token per step, and retires those that have finished. Memory for the key value cache must be allocated and freed without fragmentation. Latency must be kept low enough that a human reading the output never waits. These are the problems of systems engineering, and they are as essential to a working language model as the mathematics of the network itself.
